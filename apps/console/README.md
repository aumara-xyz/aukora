# @aukora/console — read-only operator console

A curated, public-facing window into the Aukora governed organism. It is built so an investor can understand
the organism in about five minutes: what holds authority, how memory grows and proves itself, how a self-change
is governed, what the advisory council decided, what it costs, and what is deliberately **not** turned on.

It is **read-only**. It renders a committed, deterministic `DEMO_FIXTURE` and **cannot sign, authorize, apply,
deploy, or arm anything**. It carries no private keys, tokens, prompts, private paths, or secret-shaped data.

## Two surfaces

- **`shell.html` — the Spatial shell** (primary): an inside-out trinity shell at visual parity with the donor
  Symbiote Spatial app. Three glass lanes on one navy stage, telescoping hot-corners (`[` `]` `,` `.`), and the
  ▲■○ family menu: **Triangle** AUMA LIVE · **Square** AUMLOK, AURA, SPATIAL MAP, CONSOLE, SETTINGS · **Circle**
  KNVS. CONSOLE mounts the tested panels; AUMLOK+AURA share one read-only witnessed ceremony; KNVS is a safe
  `allow-scripts`-only sandbox lab (strict CSP, draft-only). Design law: [`docs/DESIGN_CONSTITUTION.md`](docs/DESIGN_CONSTITUTION.md);
  parity: [`docs/PARITY_7090.md`](docs/PARITY_7090.md).
- **`index.html` — the flat operator console**: the same ten panels in a single scroll.

## Truth labels

- **IMPLEMENTED** — source + test + demo prove it on this path.
- **ROADMAP** — designed or wired elsewhere, not proven here.
- **UNARMED** — a capability deliberately not enabled.

Implemented vs. roadmap is never blurred. Aukora is presented as **governed · growing · remembering** — it is
**not** claimed to be alive, conscious, or self-replicating.

## The fixture is real, not invented

`public/fixture.json` (and its browser twin `public/fixture.js`) are generated by
`tooling/generate-fixture.gen.ts`, which runs the **real** `@aukora` organism (the same deterministic, offline,
in-memory reactive adapter + governed recursion the `demo:organism` proof runs) plus one **real offline** Fu
council pass, and captures their actual outputs. Every number, hash, verdict, and label comes from the canonical
packages. Regeneration is deterministic (a fixed seed instant), so it produces byte-stable diffs.

```bash
# from the repo root, once:  npm install  &&  npm run build:kernel

npm run build:kernel                          # build kernel subpath exports first
npm run fixture --workspace @aukora/console   # regenerate the fixture from the real organism
npm run test    --workspace @aukora/console   # validate the fixture + the read-only/no-secrets safety contract
npm run serve   --workspace @aukora/console   # preview at http://127.0.0.1:4173/ (zero-dependency static server)
```

The shipped console needs no server and no build: `public/index.html` loads the fixture as a plain script, so it
renders from `file://` or any static host with no network request. The preview server exists only so a reviewer
can open it locally.

## Accessibility

Semantic landmarks, a skip link, keyboard-focusable sections with an in-page section nav, visible focus rings,
a responsive single-to-multi-column grid, light/dark theming (with a `data-theme` override hook), and
`prefers-reduced-motion` support (the only animation is a load-in fade, disabled under reduced motion).
